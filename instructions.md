# Инструкция по работе с ботом (Meta-Harness)

Этот проект представляет собой Telegram-бота на базе методологии Meta-Harness для сбора инсайтов участников во время конференций, а также эндпоинты администратора для запуска опросов и аналитики.

---

## 🚀 Как запустить проект локально

### Шаг 1: Установка зависимостей
Убедитесь, что у вас установлен Node.js. В корневой папке проекта выполните:
```bash
npm install
```

### Шаг 2: Настройка переменных окружения
В файле `.env` в корне проекта уже прописаны ваши API-ключи. Укажите желаемый пароль администратора для защиты эндпоинтов:
```text
ADMIN_PASSWORD=ваш_пароль_админа
```

### Шаг 3: Запуск локального сервера
Для запуска сервера симуляции Vercel Serverless введите команду:
```bash
npm start
```
Сервер запустится на порту `3000`.

### Шаг 4: Проброс вебхука в Telegram
Так как сервер работает локально, Telegram не сможет присылать сообщения напрямую на `localhost`. Необходимо создать публичный адрес:
1. Запустите [ngrok](https://ngrok.com) (или аналогичную утилиту):
   ```bash
   ngrok http 3000
   ```
2. Скопируйте полученный HTTPS адрес (например, `https://a1b2-34-56.ngrok-free.app`).
3. Зарегистрируйте вебхук в Telegram, выполнив команду в терминале (или открыв её в браузере):
   ```bash
   curl -X POST https://api.telegram.org/bot8605173385:AAF3uaFWDGvJG7VRmRpSfYRllQvgAJlQm4E/setWebhook?url=https://a1b2-34-56.ngrok-free.app/api/webhook
   ```

## 📂 Как устроена база данных (db.json)
* По умолчанию все данные участников, сессий, блокнотов и инсайтов пишутся в локальный файл `db.json` в корневой папке.
* Вы можете открыть этот файл прямо в IDE, чтобы следить за базой в реальном времени при тестировании бота.

## 🛠 Вызовы эндпоинтов администратора

Все эндпоинты защищены паролем `ADMIN_PASSWORD` из вашего файла `.env`.

### 1. Запуск опроса по сессии (`/api/push-poll`)
Переводит участников в режим сбора инсайтов по конкретной сессии и рассылает им вопрос.

* **Метод**: `POST`
* **URL**: `http://localhost:3000/api/push-poll`
* **Заголовки**: `Content-Type: application/json`
* **Тело запроса (JSON)**:
  ```json
  {
    "session_id": "session_luksha",
    "title": "Форсайт-лекция Павла Лукши",
    "question": "Какие главные поинты вы выделите после выступления Павла Лукши о мегатрендах?",
    "admin_password": "ваш_пароль_админа"
  }
  ```

### 2. Экспорт собранных инсайтов в Excel (`/api/analytics`)
Генерирует Excel-файл со всеми ответами участников и присылает его вам в Telegram.

* **Метод**: `GET` (можно открыть прямо в адресной строке браузера)
* **URL**: `http://localhost:3000/api/analytics?action=excel&session_id=session_luksha&admin_tg_id=ID_ВАШЕГО_TELEGRAM&admin_password=ваш_пароль_админа`
  *(Вместо ID_ВАШЕГО_TELEGRAM вставьте ваш цифровой ID, получить который можно в боте `@userinfobot`).*

### 3. ИИ-Анализ трендов через DeepSeek (`/api/analytics`)
Сформирует структурированный Markdown-отчет о ТОП-5 сквозных трендах, уникальных инсайтах и облаке тегов сессии.

* **Метод**: `GET`
* **URL**: `http://localhost:3000/api/analytics?action=trends&session_id=session_luksha&admin_tg_id=ID_ВАШЕГО_TELEGRAM&admin_password=ваш_пароль_админа`

## ☁️ Развертывание в продакшн на Vercel
При деплое на Vercel Serverless локальный файл `db.json` перестанет сохранять данные из-за ограничений файловой системы. Для этого настроено автоматическое облачное переключение:
1. Загрузите проект на Vercel.
2. Впишите переменные из `.env` в разделе **Environment Variables** настроек вашего проекта на Vercel.
3. Во вкладке **Storage** вашего проекта Vercel нажмите **Create Database** и выберите **KV (Redis)**.
4. Нажмите кнопку **Connect**. Vercel автоматически добавит нужные переменные подключения.
5. Наш код автоматически обнаружит облачный Redis и переключится на него для полноценного и быстрого сохранения данных в облаке.
